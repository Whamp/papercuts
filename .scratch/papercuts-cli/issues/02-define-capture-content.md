# Define What a Capture Must Contain

Type: grilling
Status: resolved
Blocked by: 01

## Question

What minimum information must each papercut contain—including precise severity meanings, description quality, timestamp, and attribution—so humans and agents report predictably without turning the log into an issue tracker?

## Answer

Each papercut contains exactly these concepts:

1. **Captured at** — required and generated by the CLI after validation, in UTC using RFC 3339 with fractional seconds when present. The caller cannot supply or override it.
2. **Severity** — required: `low`, `medium`, or `high`.
   - `low`: an avoidable detour that did not change the approach or confidence in the result.
   - `medium`: meaningful rework, repeated attempts, a workaround, a changed approach, or reduced confidence while the task remained safely completable.
   - `high`: blocked completion, required human or environment intervention, or credible risk of an incorrect, destructive, or insecure result.
   - When several definitions apply, use the highest severity.
3. **Description** — required non-empty UTF-8 prose. Preserve internal whitespace and newlines; trim only leading and trailing whitespace. Guidance asks for one concrete friction instance: what the reporter attempted or expected, what happened, and its impact. Include the workaround or current state when known. The CLI validates presence, not prose quality.
4. **Attribution** — optional caller-supplied `reporter` and `model` labels. Store either, both, or neither. Never infer them from Git, the operating-system account, or unrelated environment state.

The selected project or global log carries scope, so entries do not duplicate it. Descriptions may mention task identifiers, commands, paths, links, or suggested improvements, but those remain prose rather than structured fields.

A capture records an observation, including non-fatal friction. It has no title, status, owner, assignee, due date, priority separate from severity, category, tags, comments, duplicate relationship, or resolution workflow. It never creates a repair ticket. Agent guidance must tell reporters not to include secrets.
