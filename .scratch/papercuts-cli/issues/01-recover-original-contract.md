# Recover the Original Tool’s Observable Contract

Type: research
Status: resolved

## Question

What command surface, file format, metadata acquisition, setup behavior, and agent guidance from Steve Ruiz’s original papercuts tool can be verified from public sources, and which details remain unknown?

## Answer

The only located first-party evidence is [Steve Ruiz’s post](https://x.com/steveruizok/status/2075303919664734295) and its [screenshot](https://pbs.twimg.com/media/HMz1tvqWoAA6wh2?format=png&name=large). They verify a tiny CLI called `papercuts`, intended for agents to record workflow frustrations they would otherwise silently push through. The screenshot shows human-readable entries with an ISO-8601 UTC timestamp, apparent model identifier, apparent user identifier, and contextual multiline prose, including non-fatal friction.

No public primary source located in this iteration establishes the command syntax, backing filename or serialization, metadata acquisition, setup/install behavior, exact agent instructions, severity model, or persistence semantics. The screenshot is presentation evidence, not a storage specification.

Full evidence, transcriptions, source checks, and unknowns: [original tool contract research](../research/original-tool-contract.md).
